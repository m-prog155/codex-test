# 4.X 实验结果与系统效果分析

## 4.X.1 实验环境与参数设置

本文车牌检测模型与车牌专用 OCR 模型的训练和验证工作均在远端 GPU 服务器上完成，运行环境为 `Ubuntu 22.04`、`Python 3.12.3`。车牌检测部分使用 `PyTorch 2.8.0+cu128` 与 `YOLO26n` 预训练权重进行迁移学习；车牌专用 OCR 部分使用 `PaddleOCR` 官方识别模型 `PP-OCRv5_mobile_rec` 作为初始化参数。远端显卡为 `RTX 5090 32GB`，能够满足本课题中检测模型训练、OCR 微调和系统联调的计算需求。

为保证检测对比实验的公平性，本文在两组检测实验中保持模型结构和主要训练策略一致，仅调整训练数据规模。检测训练参数设置为：训练轮数 `50`，输入尺寸 `640×640`，批大小 `16`。其中，quick 方案使用小规模数据集，划分为 `train 5000 / val 1000 / test 2000`；mvp 方案使用扩展数据集，划分为 `train 20000 / val 5000 / test 10000`。两组实验均以 `mAP@0.5` 和 `mAP@0.5:0.95` 作为主要评价指标，并结合 `Precision` 与 `Recall` 进行综合分析。

在 OCR 专用模型训练中，本文基于 CCPD 文件名真值构建了识别训练集，划分为 `train 20000 / val 5000 / test 10000`。识别训练轮数设置为 `75`，并在训练完成后导出 inference 模型，用于后续离线评估与系统演示联调。

从表 `4-1` 可以看出，在保持模型结构和训练策略不变的条件下，扩大训练数据规模后，车牌检测模型的整体性能得到提升。quick 方案在第 `47` 个 epoch 达到最佳性能，`mAP@0.5:0.95` 为 `0.77963`；mvp 方案在第 `46` 个 epoch 达到最佳性能，`mAP@0.5:0.95` 提升至 `0.78672`。与 quick 方案相比，mvp 方案在该指标上提升了 `0.00709`，相对提升约为 `0.91%`。

## 4.X.4 OCR 小样本评估结果分析

为避免仅凭样例现象判断 OCR 优化效果，本文首先基于 CCPD 文件名真值开展了 OCR 小样本对照评估。该评估直接使用 CCPD 真值框裁出的车牌图像，在同一批样本上比较“优化前 OCR”和“基于规则稳定化后的 OCR”的输出差异，从而尽量减弱检测误差对结论的干扰。

本次评估使用 `D:\plate_project\CCPD2019\splits\test.txt` 作为数据来源，按照默认抽样策略设置 `seed = 42`、`per_subset = 10`。设计上原计划覆盖 `ccpd_base`、`ccpd_blur`、`ccpd_db`、`ccpd_rotate`、`ccpd_tilt`、`ccpd_weather` 和 `ccpd_challenge` 七个子集，但当前划分文件中仅包含 `ccpd_blur`、`ccpd_db`、`ccpd_rotate`、`ccpd_tilt` 和 `ccpd_challenge` 五个目标子集，因此最终有效样本数为 `50`。评估指标包括整牌完全正确率、字符级准确率和无结果样本数。

### 表 4-2 OCR 小样本对照评估结果

| 模式 | 样本数 | 整牌完全正确数 | 整牌完全正确率 | 字符正确数 / 总字符数 | 字符级准确率 | 无结果样本数 | 无结果比例 |

从表 `4-2` 可以看出，在本次 `50` 张 CCPD 小样本评估中，基于规则的 OCR 稳定化并未表现出整体识别精度提升。与 baseline 相比，stabilized 模式的整牌完全正确率由 `10.0%` 降至 `6.0%`，字符级准确率由 `30.33%` 降至 `15.33%`，同时无结果样本数由 `0` 上升到 `25`。这说明当前这轮规则稳定化策略的主要效果并不是“识别更准”，而是“输出更保守”。

## 4.X.5 车牌专用 PaddleOCR 对比评估

由于前述基于规则的 OCR 稳定化策略未在小样本评估中体现出整体精度提升，本文进一步采用 CCPD 车牌裁剪图像构建识别训练集，并对 PaddleOCR 识别模型进行了车牌任务微调。训练数据来源于远端 `ccpd_yolo_mvp` 数据集图像，按 `train 20000 / val 5000 / test 10000` 划分生成识别样本；训练时以 `PP-OCRv5_mobile_rec` 预训练模型为初始化参数，训练轮数设置为 `75`，最终在第 `74` 个 epoch 取得最优验证指标 `acc = 0.9856`。

在完成模型导出后，本文进一步在完整 `10000` 张测试样本上比较了“通用 PaddleOCR 识别器”和“车牌专用 PaddleOCR 识别器”的表现。评估指标仍采用整牌完全正确率、字符级准确率和无结果样本数。

### 表 4-3 通用 OCR 与车牌专用 PaddleOCR 对比结果

| 模式 | 样本数 | 整牌完全正确数 | 整牌完全正确率 | 字符正确数 / 总字符数 | 字符级准确率 | 无结果样本数 | 无结果比例 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |

从表 `4-3` 可以看出，车牌专用 PaddleOCR 在本次完整测试集评估中取得了更好的识别效果。与通用 OCR 相比，specialized 模式的整牌完全正确率由 `2.52%` 提升至 `14.80%`，提升了 `12.28` 个百分点；字符级准确率由 `31.32%` 提升至 `38.28%`，提升了 `6.96` 个百分点。该结果说明，当识别模型针对车牌字符分布和任务场景进行专门训练后，整体识别能力能够得到实质性改善。

与此同时，specialized 模式的无结果比例由 `28.20%` 上升到 `32.79%`，增加了 `4.59` 个百分点。这表明专用 OCR 在提升整牌和字符准确率的同时，仍然保留了一定的保守输出倾向。结合前述视频样例结果可以看出，最终系统在困难样本上仍可能给出 `null` 或不完全正确的文本，但从完整测试集的真实对比结果来看，专用 PaddleOCR 已经明显优于通用 OCR，因此本文最终将其作为系统演示与论文实验中的主识别方案。
