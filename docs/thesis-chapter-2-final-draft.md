# 第2章 国内外研究现状与相关技术分析

## 2.1 车辆目标检测研究现状

近几年，基于深度学习的目标检测方法已经成为交通场景视觉感知任务的主流技术路线。游昊等[1]和董红召等[2]对相关研究进行了综述，指出车辆检测任务已经从早期依赖手工特征和传统分类器的方法，逐步转向以卷积神经网络为基础的端到端检测框架。其中，YOLO 系列模型由于推理速度快、结构清晰、工程部署成本相对较低，在交通目标检测、车辆检测和道路场景感知中应用尤为广泛。

围绕车辆检测任务，近两年的国内研究主要集中在多尺度特征融合、轻量化结构改进、注意力机制增强以及复杂场景适配等方向。李明芳[3]针对自动驾驶场景提出 DMP-YOLO，通过多尺度特征处理提升目标检测性能；许德刚等[4]围绕城市车辆目标检测对 YOLOv8 结构进行改进，以提升复杂城市场景中的检测效果；刘佳琳和殷丽凤[6]、蒋康等[9]则分别从改进 YOLOv8 检测结构的角度出发，研究行人和车辆检测、轻量化车辆检测等问题。刘超阳等[15]提出轻量级车辆检测模型，说明在资源受限场景下，轻量化仍然是车辆检测研究的重要方向。

在复杂交通环境下，研究重点进一步从“能否检测到”转向“在遮挡、尺度变化、小目标和复杂背景条件下能否稳定检测”。张浩晨等[11]面向复杂交通场景提出 YOLO-NPDL 检测方法，牟俊宇等[12]将注意力机制引入 YOLOv7 用于车辆识别检测，徐慧智等[14]则聚焦高速公路场景中的小目标检测问题。这些工作表明，复杂交通场景下的车辆检测已经不再只是常规目标检测问题，而是需要同时兼顾实时性、轻量化和复杂样本鲁棒性的综合任务。

从国外研究看，车辆与车牌相关检测任务更强调系统整体性能与应用落地。Agarwal 和 Bansal[19]基于 YOLO World 研究车牌检测与识别问题，Al-Hasan 等[20]提出基于改进 YOLOv8 的自动车牌识别系统，Sonnara 等[16]则将深度学习车牌识别系统应用于边缘设备环境，关注实时性与部署效率。可以看出，国外研究在保持检测精度的同时，更关注端到端系统在真实场景中的运行效率与工程适配能力。

总体而言，当前车辆目标检测研究呈现出两个明显趋势：一是以 YOLO 为代表的单阶段检测框架仍然占据主流地位；二是研究重点从单纯提高检测精度，逐步扩展到轻量化、复杂场景鲁棒性以及系统级实时部署能力。上述研究为本文车辆检测模块的技术选型提供了直接参考。

## 2.2 车牌检测与识别研究现状

与车辆目标检测相比，车牌检测与识别任务通常需要同时处理目标定位和字符识别两个层面的问题，因此整体难度更高。尤其在动态交通场景中，车牌目标往往具有尺寸小、角度变化大、模糊遮挡明显等特点，这使得车牌识别对检测精度、裁剪质量和识别模型能力都提出了更高要求。

近年来，国内在车牌检测与识别方面的研究多采用“检测 + 识别”的两阶段技术路线。杨子奕等[5]基于 YOLOv5 开展动态车牌识别研究，并结合残差网络优化识别效果；宋建辉等[7]将改进 YOLOv8 与 LPRNet 结合，用于车牌识别任务。此类研究说明，在复杂场景下，将车牌定位与字符识别分解为两个相对独立的子任务，仍然是当前较为稳妥且工程上可操作的方案。

从识别模型角度看，车牌识别已经逐渐从“通用 OCR 直接迁移”转向“车牌任务专用识别”。一方面，车牌字符分布具有较强的领域特性，字符长度、字符种类和版式结构都明显不同于自然场景文本；另一方面，模糊、倾斜、低照度和重复字符等问题也使通用 OCR 难以直接取得稳定效果。Cui 等[17]发布的 PaddleOCR 3.0 技术报告表明，OCR 框架正在朝着更强的训练能力和更完善的工程生态方向发展，这为特定领域文本识别模型的微调提供了便利。Sonnara 等[16]的研究则表明，针对车牌识别任务构建专用深度学习识别方案，有助于兼顾识别准确率和部署效率。

此外，数据集构建对车牌识别性能具有直接影响。Agrawal[18]提出全球车牌数据集，反映出不同国家和地区的车牌字符形式、布局规则和图像质量差异较大。这说明车牌识别模型通常具有较强的数据依赖性，若想在具体场景下取得更好效果，往往需要结合目标数据集进行专门训练或微调。对本课题而言，使用基于 CCPD 文件名真值构造的车牌识别样本对 PaddleOCR 进行专用微调，正是基于这一研究规律所作出的技术选择。

综合国内外研究可以看出，车牌识别任务的研究重点已经从传统字符分割与模板匹配，逐渐过渡到深度学习驱动的端到端识别或“检测 + 专用识别”融合方案。对于强调系统可运行、可演示和可验证的毕业设计而言，后一种路线在工程上更稳定，也更便于进行误差分析和模块替换。

## 2.3 本文相关技术路线分析

结合上述研究现状可以发现，当前交通场景视觉感知系统通常围绕三个核心问题展开：一是如何在复杂场景中稳定完成车辆和车牌目标定位；二是如何在模糊、倾斜和小目标条件下提高字符识别准确率；三是如何在保证系统可运行性的前提下控制开发复杂度和实验成本。

在检测模块方面，YOLO 系列模型已经在大量研究中证明了其在交通场景中的有效性[1-4][6][9][11][12][14][15][19][20]。因此，本文在车辆检测和车牌检测任务上继续采用 YOLO 作为统一检测后端，既符合现有研究主流，也有利于保证系统实现的简洁性和实验复现性。

在 OCR 模块方面，现有研究一方面表明专用车牌识别模型通常优于直接使用通用 OCR，另一方面也说明成熟 OCR 框架在训练、导出和部署方面具有较好的工程基础[7][16][17]。考虑到本课题已经形成了 Python 端完整推理流程，并且最终目标是完成一个可演示、可实验验证的 PC 端系统，而不是追求独立设计全新识别网络，因此本文选择以 PaddleOCR 作为识别框架基础，并结合车牌裁剪数据构建专用识别模型。这一路线既保留了通用 OCR 框架的工程优势，也吸收了“任务专用化识别”的研究思路。

从系统实现角度看，本文技术路线最终确定为“YOLO 检测 + 车牌专用 PaddleOCR 识别”。该方案的优点主要体现在以下三个方面。第一，模块边界清晰，车辆检测、车牌检测和字符识别之间职责明确，便于分别调试与替换。第二，技术栈统一，能够在 Python 环境中完成训练、推理、评估和演示闭环，适合毕业设计周期内实现。第三，实验上具有可比较性，既可以保留通用 OCR 作为 baseline，也可以通过专用 PaddleOCR 微调获得更符合车牌任务特点的识别模型。

需要说明的是，本文并不试图提出新的车辆检测网络或新的车牌识别理论模型，而是基于现有成熟研究成果完成系统集成与针对性优化。从课题定位来看，这一选择更符合应用型毕业设计的目标，即在合理范围内完成可运行系统的设计、实现和实验验证。

## 2.4 本章小结

本章围绕车辆目标检测、车牌检测与识别两个方面，对近几年国内外相关研究进行了梳理。现有研究表明，YOLO 系列模型在交通场景目标检测中具有较高的实用价值，而车牌识别任务则越来越强调专用识别模型和任务数据集的重要性。

基于上述研究现状，本文最终选择“YOLO 检测 + 车牌专用 PaddleOCR 识别”作为系统总体技术路线。该路线既符合当前相关研究的发展趋势，也兼顾了系统实现难度、实验可操作性和毕业设计的实际目标，为后续系统设计与实验分析奠定了理论与技术基础。
